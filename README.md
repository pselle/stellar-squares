# Stellar Squares NFT Collection

[![Built with Scaffold Stellar](https://img.shields.io/badge/Built_with-Scaffold_Stellar-blue?style=for-the-badge)](https://scaffoldstellar.org)
[![Powered by OpenZeppelin](https://img.shields.io/badge/Powered_by-OpenZeppelin_Smart_Contracts-green?style=for-the-badge)](https://github.com/OpenZeppelin/stellar-contracts)

**A showcase project demonstrating modern NFT development on Stellar using Scaffold Stellar and OpenZeppelin contracts**

## 🎨 About

Stellar Squares is a generative art NFT collection inspired by Georg Nees' "Schotter" featuring geometric squares created through controlled randomness. This project serves as a complete example of:

- **Full-stack dApp architecture** built with [Scaffold Stellar](https://scaffoldstellar.org)
- **Modern NFT development** using audited [OpenZeppelin Stellar contracts](https://github.com/OpenZeppelin/stellar-contracts)
- **Smart contract integration** with seamless frontend experience
- **Future-ready deployment** designed to leverage the (coming soon!) [Stellar Registry](https://testnet.rgstry.xyz/) in upcoming versions.

Each piece in the collection is a unique NFT generated by [algorithmic art](https://github.com/pselle/schotter-squares), showcasing the intersection of code, mathematics, and visual aesthetics on the Stellar blockchain.

## ✨ Features

- 🖼️ **NFT Gallery**: Browse and discover unique generative art pieces
- 💳 **Direct Purchase**: Buy NFTs directly from the gallery with wallet integration
- 🔗 **Multi-wallet Support**: Connect with various Stellar-compatible wallets
- 🎲 **Generative Art**: Each NFT is uniquely generated using controlled randomness algorithms
- 🔒 **Audited Contracts**: Built on OpenZeppelin's battle-tested smart contracts
- 🚀 **Production Ready**: Deployable to mainnet

## 🏗️ Architecture

This project demonstrates a complete NFT gallery built on Stellar:

- **Frontend**: React + TypeScript with Vite, featuring wallet integration and responsive design
- **Smart Contracts**:
  - Gallery contract for NFT management and sales
  - OpenZeppelin NFT contract for secure token operations
- **Blockchain**: Stellar network with native asset (XLM) integration
- **Deployment**: Vercel with automated CI/CD via GitHub Actions

## 🚀 Developer Quick Start

### Prerequisites

Before getting started, make sure you’ve met the requirements listed in the [Soroban documentation](https://developers.stellar.org/docs/build/smart-contracts/getting-started/setup) and that the following tools are installed :

- [Rust](https://www.rust-lang.org/tools/install)
- [Cargo](https://doc.rust-lang.org/cargo/) (comes with Rust)
- Rust target: install the compilation target listed in the [Soroban setup guide](https://developers.stellar.org/docs/build/smart-contracts/getting-started/setup)
- [Node.js & npm](https://nodejs.org/en/download/package-manager) (v22, or higher)
- [Stellar CLI](https://github.com/stellar/stellar-core)
- [Scaffold Stellar CLI Plugin](https://github.com/theahaco/scaffold-stellar)

### Installation

1. **Clone and install dependencies:**

   ```bash
   git clone <repository-url>
   cd important_diagrams
   npm install
   ```

2. **Set up local Stellar environment:**

   ```bash
   stellar network use local
   stellar contract upload --wasm contracts/squares-gallery/fixtures/nft_sequential_minting_example.wasm
   # Copy the returned hash
   ```

3. **Update configuration:**
   Edit `environments.toml` with your WASM hash:

   ```toml
   [development.contracts.squares_gallery]
   ...
   constructor_args = "--owner me --nft_wasm_hash YOUR_WASM_HASH_HERE --xlm_sac $(stellar contract id asset --asset native)"
   ```

4. **Start development server:**
   ```bash
   npm run start
   ```

The application will be available at `http://localhost:5173`

## 📦 Deployment

### Production Deployment

The project is deployed to Vercel as a static build with automated CI/CD through GitHub Actions.

**Deploy to staging (testnet):**

```bash
npm run build:staging
npx vercel build
npx vercel --prebuilt
```

**Deploy to production (mainnet):**

```bash
npm run build
npx vercel build --prod
npx vercel --prebuilt --prod
```
